## `$ ls -la ./projects`

### 📦 `ci-cd-pipeline/` &nbsp;—&nbsp; *Jenkins · Terraform · AWS EC2 · SonarQube*
> Push code → quality gate → infra provisioned → deployed. No humans in the loop.

Built a zero-touch delivery pipeline for a React app. Every commit triggers Jenkins, which runs SonarQube checks — fail the gate, nothing ships. Pass, and Terraform spins up EC2, the build lands on it, live. Reproducible, observable, no manual SSH.

[![Repo](https://img.shields.io/badge/github-ci--cd--pipeline-2d2d2d?style=flat-square&logo=github&logoColor=white)](https://github.com/)

---

### 📦 `node-ansible-deploy/` &nbsp;—&nbsp; *Ansible · Node.js · AWS EC2 · PM2*
> One playbook. Code pulled, deps installed, process supervised. Done.

Ansible playbook that provisions and deploys a Node.js app on AWS EC2 end to end — pulls from GitHub, installs, starts with PM2 (so it survives reboots), and locks down the security group. Repeatable on any fresh EC2 instance. No drift, no guesswork.

[![Repo](https://img.shields.io/badge/github-ansible--deploy-2d2d2d?style=flat-square&logo=github&logoColor=white)](https://github.com/)

---

### 📦 `blogverse/` &nbsp;—&nbsp; *React · Node.js · Express · MongoDB*
> Full-stack blog with real auth, real roles, real backend.

Blogverse isn't a tutorial CRUD app — it has secure authentication, role-based access control (authors vs admins, enforced server-side), and a responsive React frontend. Backend owns the permission model. Frontend just reflects it.

[![Repo](https://img.shields.io/badge/github-blogverse-2d2d2d?style=flat-square&logo=github&logoColor=white)](https://github.com/)
